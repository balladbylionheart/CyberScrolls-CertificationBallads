**SC-900 | Security Acronyms**

---

## 📖 Concept Overview

Security certifications introduce many acronyms that represent important security concepts and technologies.

Understanding what these acronyms stand for—and what they do—is essential for success on the SC-900 exam.

This scroll reinforces foundational security terminology through repetition and rhythm, making common abbreviations easier to remember.

---

## 🔤 Acronyms Covered

### 👤 RBAC
**Role-Based Access Control**

RBAC assigns permissions based on a user's role.

Key concepts:

- Least privilege
- Access based on job responsibilities
- Simplified permission management

---

### 🔐 MFA
**Multifactor Authentication**

MFA requires users to provide more than one form of verification before gaining access.

Key concepts:

- Multiple authentication factors
- Stronger identity verification
- Improved account security

---

### 🌐 DDoS
**Distributed Denial-of-Service**

A DDoS attack attempts to overwhelm a service with traffic, preventing legitimate users from accessing it.

Key concepts:

- Service disruption
- Traffic flooding
- Perimeter protection

---

### 🔗 NSG
**Network Security Group**

NSGs control inbound and outbound network traffic through allow and deny rules.

Key concepts:

- Network filtering
- Traffic control
- Segmentation support

---

### ⚙️ XSS
**Cross-Site Scripting**

XSS is an attack that injects malicious scripts into trusted websites or applications.

Key concepts:

- Script injection
- Browser-based attacks
- Secure input handling

---

### 🗄️ SQL Injection
**Structured Query Language (SQL) Injection**

SQL Injection occurs when malicious input is used to manipulate database queries.

Key concepts:

- Database attacks
- Input validation
- Secure coding practices
