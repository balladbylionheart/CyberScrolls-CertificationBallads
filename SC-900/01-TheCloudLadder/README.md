# 🎵 Scroll I — The Cloud Ladder  
**SC-900 | Shared Responsibility Model**

---

## 📖 Concept Overview

The **Shared Responsibility Model** explains how security responsibilities are divided between the **cloud provider** and the **customer**.

As organizations move from **on-premises → IaaS → PaaS → SaaS**, the provider takes on more responsibility for managing the underlying infrastructure.

However, the customer always remains responsible for key elements such as **data, access, and configuration**.

---

## ☁️ Cloud Service Models Explained

### 🏢 On-Premises
You are responsible for everything:

- Physical datacenter  
- Servers and networking  
- Operating systems  
- Applications  
- Identity and data  

---

### 🧱 IaaS (Infrastructure as a Service)

The provider manages:
- Datacenter  
- Physical hardware  
- Networking infrastructure  

You manage:
- Operating systems  
- Applications  
- Network configuration  
- Identity and data  

---

### ⚙️ PaaS (Platform as a Service)

The provider manages:
- Infrastructure  
- Operating system  
- Runtime environment  

You manage:
- Applications  
- Configuration  
- Identity and access  
- Data  

---

### 💻 SaaS (Software as a Service)

The provider manages:
- Full application  
- Infrastructure and platform  

You manage:
- User access  
- Identity and authentication  
- Data within the application  

---

## 📈 The Cloud Ladder

As you move up the cloud models:

- **IaaS → You manage more**
- **PaaS → Shared management**
- **SaaS → Provider manages more**

👉 The higher you go, the less infrastructure you manage.

---

## 🧠 Key Takeaway

You can delegate infrastructure to the cloud provider,  
but you never delegate responsibility for:

- Data  
- Identity and access  
- Configuration
